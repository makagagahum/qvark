The software license lets you use, study, modify, and redistribute the code under its terms. It does not grant permission to use Qorx marks in a way that implies endorsement, official status, ownership, or affiliation with Marvin Sarreal Villanueva.

## Allowed Uses

- Refer to the original project as Qorx.
- Link to the repository, DOI, papers, and benchmarks.
- State that your project is a fork of Qorx if that is true.
- Use the name in compatibility statements, such as "works with Qorx."

## Not Allowed Without Written Permission

- Present a modified fork, hosted service, binary, or commercial offering as the official Qorx project.
- Use Qorx marks in a way that creates confusion about source, sponsorship, or endorsement.
- Remove attribution from the scholarly article, citation metadata, or project notices.
- Use Qorx logos or identity marks as the primary branding for an unrelated product.

## Fork Naming

If you distribute a modified fork, use a name that clearly distinguishes it from the official Qorx project unless you have written permission.

This policy is not a trademark registration notice. Trademark registration, if pursued, is a separate legal process.
